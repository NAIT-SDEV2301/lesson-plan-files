LINQ Quick Reference Cheat Sheet

Common LINQ operators for collections and EF Core queries

LINQ Overview
  • LINQ = Language Integrated Query ‚Äî query data using C# syntax.
  • Works with IEnumerable<T> (in-memory) and IQueryable<T> (database via EF Core).
  • Two styles:
    • Query syntax: from b in books where b.Pages > 300 select b;
    • Method syntax: books.Where(b => b.Pages > 300).Select(b => b);

1Ô∏è‚É£ Filtering and Projection

Function Description Example
Where() Filter elements by condition books.Where(b => b.InStock)
Select() Transform each element into a new form books.Select(b => b.Title)
SelectMany() Flatten nested collections orders.SelectMany(o => o.Items)
Distinct() Remove duplicates genres.Distinct()

2Ô∏è‚É£ Sorting

Function Description Example
OrderBy() Sort ascending books.OrderBy(b => b.Title)
OrderByDescending() Sort descending books.OrderByDescending(b => b.Pages)
ThenBy() Secondary ascending sort books.OrderBy(b => b.Genre).ThenBy(b => b.Title)

3Ô∏è‚É£ Grouping and Aggregation

Function Description Example
GroupBy() Group by key selector books.GroupBy(b => b.Genre)
Count() Count items (optional filter) books.Count(b => b.InStock)
Sum(), Average(), Max(), Min() Aggregate numeric values orders.Sum(o => o.Total)

4Ô∏è‚É£ Set Operations

Function Description Example
Union() Combine unique elements from two sequences a.Union(b)
Intersect() Common elements of both sequences a.Intersect(b)
Except() Elements in first sequence not in second a.Except(b)

5Ô∏è‚É£ Joining and Navigation (EF Core)

Pattern Usage Example
Join() Join two collections on key orders.Join(customers, o => o.CustomerId, c => c.Id, (o, c) => new { o.Id, c.Name })
Navigation Property (EF) Use related entity directly db.Products.Select(p => new { p.ProductName, p.Category.CategoryName })

6Ô∏è‚É£ Execution & Conversion

Function Purpose Example
ToList() Execute and convert to list var list = query.ToList();
First(), FirstOrDefault() Get first element (or default/null) var first = books.FirstOrDefault();
Single(), SingleOrDefault() Expect exactly one element var result = users.Single(u => u.Id == 10);
Any(), All() Check existence or condition books.Any(b => b.InStock)

7Ô∏è‚É£ EF Core-Specific Extensions

  • ToQueryString() ‚Üí View SQL generated by a query.
  • AsNoTracking() ‚Üí Read-only queries (improves performance).
  • Include() ‚Üí Eagerly load related data.
  • FromSqlRaw() ‚Üí Run raw SQL queries.
  • Take() / Skip() ‚Üí Implement pagination.
üí° Tips:
  • LINQ to Objects executes in memory; LINQ to Entities executes on SQL Server.
  • Use ToList() only when you actually need the results (forces execution).
  • Keep filters (Where) and projections (Select) before ToList() for efficiency.

8Ô∏è‚É£ Predicate Logic & Boolean LINQ Operators

These LINQ functions evaluate conditions across collections and return true / false results.

Method Purpose Example Result
Any() Returns true if any element exists or satisfies a condition. books.Any(b => b.InStock) True if at least one book is in stock.
All() Returns true if all elements satisfy a condition. books.All(b => b.Pages > 100) True if every book has more than 100 pages.
Contains() Checks whether a sequence contains a given value. genres.Contains("Fantasy") True if ‚ÄúFantasy‚Äù appears in the list.
SequenceEqual() Compares two sequences element-by-element for equality. a.SequenceEqual(b) True if both lists contain identical elements in order.
üí° Tips:
  • Any() stops as soon as a match is found ‚Äî more efficient than Count() > 0.
  • All() short-circuits on the first false element.
  • In EF Core, Any() and All() translate to SQL EXISTS and NOT EXISTS queries.
  • Contains() becomes a SQL IN clause when used on database queries.
